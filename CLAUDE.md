# 데이터 과학 시리즈 — 유지보수 플레이북 (후임 모델용 진입점)

> 고등 '데이터 과학' 4단원 학습 페이지. **라이브 배포 중**: https://jungbucks.github.io/data-science/ (정글 앱스토어에도 노출).
> 디자인·팔레트 규칙은 `DESIGN_SPEC.md`(시리즈 헌법 — 필독), 이 문서는 작업 규칙과 절차.

## 구조

- `index.html` 허브 + `unit1~4.html` (각 단원 = 단일 파일 완결, 오프라인 더블클릭 동작).
- 순수 HTML/CSS/JS. CDN·웹폰트·이미지·localStorage 없음(교실 공용 PC 전제).
- 컴포넌트 문법(card/chip/tip/warn/demo/퀴즈 엔진)은 unit1.html이 원본 — 새 요소는 기존 모방 우선.

## 절대 규칙

1. **DESIGN_SPEC의 팔레트는 dataviz 검증 완료값** — 새 hex 발명 금지. 단원 액센트는 `--ua` 3종 토큰으로만.
2. **차트 규칙**: 한 측정값 막대 = 단일 색조+강조 1개(막대마다 다른 색 금지), 다계열만 5색 고정 순서, 값 라벨은 잉크색.
3. **성취기준 코드([12데과…])를 페이지에 표기하지 않는다** (사용자 방침). 내용 매핑은 DESIGN_SPEC 표 참조.
4. 수치가 들어간 데모·본문은 **반드시 검산** (이상치 평균 336.2, 향상도 1.33 등 — 전부 실제 계산과 일치해야 함).
5. **배포 중인 사이트다** — push는 즉시 학생에게 노출. 커밋 후 push 전 사용자 확인이 기본, 사용자가 반영 지시한 수정은 push까지.
6. 톤: "~다/~보자", 학교 생활 비유(매점·급식·도서관), 이모지는 제목·포인트만. 15px 미만 본문 금지, reduced-motion 대응.

## 작업 절차

1. 수정 → 헤드리스 검증: `msedge --headless=new --virtual-time-budget=6000 --dump-dom <파일>` 로 JS 실행 결과(데모 요소 수·계산값) 확인. 시각 변경 시 `--screenshot`.
2. 커밋(한국어, 무엇을·왜) → push(규칙 5) → 사이트에서 200 확인.
3. 단원 내용 변경 시 DESIGN_SPEC 상태 표·메모리(data-science-series) 갱신.
